# 视频画质技术指南：SDR 与 HDR 详解

本指南详细解释了 SDR、HDR、HDR10、HDR10+ 和杜比视界的区别，帮助理解视频画质技术。

---

## 1. SDR（Standard Dynamic Range）标准动态范围

### 什么是 SDR？
SDR 是传统的视频显示标准，使用 **8-bit 色彩深度**，亮度范围有限。

### 技术规格
| 参数 | 数值 |
|------|------|
| 色彩深度 | 8-bit (256 级) |
| 亮度范围 | 0.1 - 100 nits |
| 色域 | BT.709 (Rec.709) |
| 对比度 | ~3000:1 |

### 特点
- ✅ 兼容性最广，所有设备都支持
- ✅ 文件体积较小
- ❌ 亮部和暗部细节容易丢失
- ❌ 色彩范围有限

---

## 2. HDR（High Dynamic Range）高动态范围

### 什么是 HDR？
HDR 是一种增强显示技术的统称，相比 SDR 具有 **更高的亮度范围** 和 **更丰富的色彩**。

### 核心改进
| 改进项 | SDR | HDR |
|--------|-----|-----|
| 亮度范围 | 100 nits | 1000 - 10000 nits |
| 色彩深度 | 8-bit | 10-bit / 12-bit |
| 色域 | BT.709 | BT.2020 |
| 灰阶级数 | 256 | 1024 - 4096 |

### HDR 带来的视觉提升
```
SDR 画面:   ████████████████░░░░░░░░░░
           ↑ 100 nits 亮度上限

HDR 画面:  ████████████████████████████
           ↑ 1000+ nits 亮度上限
           更多的暗部细节和亮部细节
```

---

## 3. HDR10

### 概述
HDR10 是 **开放、免费** 的 HDR 标准，被广泛采用。

### 技术规格
| 参数 | 数值 |
|------|------|
| 色彩深度 | 10-bit |
| 亮度范围 | 1000 - 4000 nits (元数据静态) |
| 色域 | BT.2020 |
| 元数据 | 静态（整个影片统一） |
| 许可费 | 免费 |

### 特点
- ✅ 开放标准，无需授权费
- ✅ 蓝光、流媒体、游戏广泛支持
- ✅ 兼容性最好
- ⚠️ 元数据静态，无法针对场景优化
- ⚠️ 不支持动态场景优化

### 支持设备
- Sony、Samsung、LG、Hisense 等电视
- Xbox、PlayStation 游戏主机
- Apple TV、Fire TV Stick
- 大多数现代智能电视

---

## 4. HDR10+

### 概述
HDR10+ 是 HDR10 的 **增强版本**，由 Samsung 开发并开放。

### 技术规格
| 参数 | HDR10 | HDR10+ |
|------|-------|--------|
| 色彩深度 | 10-bit | 10-bit |
| 亮度范围 | 1000-4000 nits | 1000-4000 nits |
| 元数据 | 静态 | **动态**（逐场景/逐帧） |
| 色域 | BT.2020 | BT.2020 |
| 许可费 | 免费 | 免费（对内容创作者） |

### 核心优势：动态元数据

```
HDR10 (静态元数据):
  影片整体使用同一亮度设置
  场景A → 1000 nits
  场景B → 1000 nits  ← 可能过于明亮或过于暗淡

HDR10+ (动态元数据):
  每个场景独立优化
  场景A → 800 nits   ← 适配该场景
  场景B → 1200 nits  ← 适配该场景
```

### 特点
- ✅ 向后兼容 HDR10
- ✅ 动态元数据优化每个场景
- ✅ 免费授权
- ❌ 生态规模小于杜比视界
- ❌ 部分设备不支持

### 支持设备
- Samsung、Toshiba、Panasonic 电视
- Amazon Fire TV
- 部分流媒体内容

---

## 5. Dolby Vision（杜比视界）

### 概述
杜比视界是 ** Dolby 开发的顶级 HDR 方案**，提供最精细的画质优化。

### 技术规格
| 参数 | 数值 |
|------|------|
| 色彩深度 | 12-bit (理论上) |
| 亮度范围 | 最高 10000 nits |
| 色域 | BT.2020、P3 |
| 元数据 | **动态**（逐场景/逐帧） |
| 许可费 | 付费 |

### 核心优势

#### 1. 逐帧动态元数据
```
杜比视界: 精确到每一帧的亮度优化
          能够完美呈现导演意图
```

#### 2. 场景重建（Scene-by-Scene）
- 分析每个场景的完整亮度分布
- 实时调整显示参数
- 消除"亮度突变"带来的不适

#### 3. 12-bit 色深处理
```
8-bit:   256 色阶 / 通道
10-bit:  1024 色阶 / 通道
12-bit:  4096 色阶 / 通道

12-bit 带来更平滑的色彩过渡，消除色带(Banding)
```

### 特点
- ✅ 最高画质，业界标准
- ✅ 精确的动态元数据
- ✅ 12-bit 色深支持
- ✅ 严格的认证流程
- ❌ 需要付费授权
- ❌ 设备认证成本高

### 支持设备
- LG、Sony、Visio、Hisense 高端电视
- Apple TV 4K
- iPhone 12+、iPad Pro
- Xbox Series X|S
- 蓝光碟、流媒体（Netflix、Disney+）

## 视频制作与分发流程

```
1. 母片制作 (12-bit RAW)
   ↓
2. 调色与格式转换
   ├── → SDR 版本 (H.264/AVC)
   ├── → HDR10 版本 (H.265/HEVC)
   ├── → HDR10+ 版本 (H.265/HEVC + 动态元数据)
   └── → Dolby Vision 版本 (H.265/HEVC + 杜比元数据)
   ↓
3. 流媒体/蓝光分发
   ↓
4. 用户设备解码播放
```

## 常见问题

### Q1: 我的设备支持哪种 HDR？
检查电视说明书或规格表：
- "HDR10" = 基础 HDR 支持
- "HDR10+, Dolby Vision" = 高级 HDR 支持

### Q2: HDR10+ 和杜比视界哪个更好？
- **画质**：杜比视界 ≈ HDR10+（杜比略优）
- **兼容性**：HDR10+ > 杜比视界
- **设备覆盖**：两者都在增长

### Q3: 观看 HDR 内容需要什么？
1. 支持 HDR 的显示设备（电视/显示器）
2. HDR 兼容的播放器
3. HDR 格式的视频源

### Q4: 为什么有些 HDR 效果不明显？
- 设备峰值亮度不足（需要 600+ nits）
- 环境光线过亮
- 片源质量差异

---

## 总结

- **SDR**: 传统标准，兼容性最好
- **HDR10**: 入门级 HDR，开放免费
- **HDR10+**: 增强版 HDR10，动态优化
- **Dolby Vision**: 顶级 HDR，付费授权

选择建议：
- 预算有限 → HDR10 电视足够
- 追求画质 → 选择 HDR10+ 或 Dolby Vision 认证设备
- 内容为王 → 根据常用平台选择（Netflix → Dolby Vision，Amazon → HDR10+）
